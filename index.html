monedux-tracker.vercel.app
Ver proyecto

Monedux Tracker

App de finanzas personales con dashboard, transacciones, presupuestos, metas de ahorro, préstamos y reportes en PDF.

ReactNestJSPostgreSQL
Objetivo

Centralizar la gestión financiera personal en un panel claro y seguro: registrar movimientos, controlar presupuestos, planificar ahorros y consultar el estado del dinero sin depender de hojas de cálculo.

Cómo funciona

El usuario se registra, configura categorías y define presupuestos por periodo. Desde el dashboard visualiza balance, ingresos, gastos y gráficos; registra transacciones, metas de ahorro y préstamos. Recibe alertas por correo (presupuesto excedido, metas alcanzadas) y puede exportar reportes en PDF.

Stack y arquitectura

Frontend: React 19, TypeScript, Vite, Tailwind CSS v4, React Router, TanStack Query, Recharts y jsPDF.
Backend: NestJS con API REST, Prisma ORM y PostgreSQL; autenticación JWT, validación con class-validator y envío de correos con Nodemailer.

Chronos — Calendario Inteligente

Calendario web con clima, festivos de Colombia, fases lunares, imagen del cosmos y eventos personales en múltiples vistas.

ReactFastAPITailwind
Objetivo

Crear un calendario informativo y visualmente distinto al convencional: combinar fechas, contexto astronómico, clima y cultura local en una sola experiencia web inmersiva.

Cómo funciona

Ofrece vista de mes, año y agenda con festivos colombianos, fase lunar y clima según ubicación. Muestra la imagen del cosmos del día (NASA), permite crear eventos propios guardados en el navegador y cambiar entre 7 temas visuales (Cosmos, Aurora, Sakura, etc.).

Stack y arquitectura

Frontend: React 19, TypeScript, Vite y Tailwind CSS v4; componentes modulares para calendario, panel lateral y modales; eventos en localStorage.
Backend: Python con FastAPI y Uvicorn; servicios para clima (Open-Meteo), festivos (Nager.Date), fases lunares, NASA Image Library y geocodificación.

talonario-gamma.vercel.app
Ver proyecto

Talonario

Sistema de venta de lotería y rifas con tablero de 100 boletas, registro de participantes, sorteo justo y exportación de reportes en PDF.

Vue 3PiniaVite
Objetivo

Digitalizar la gestión de un talonario de lotería o rifa: vender y reservar las 100 boletas (del 00 al 99), llevar control de participantes y pagos, y facilitar el sorteo y el registro impreso de ventas.

Cómo funciona

El vendedor configura premio, precio, lotería y fecha del sorteo. Desde el tablero vende o reserva boletas con datos del participante (departamento y ciudad vía API Colombia), marca pagos y ganadores, consulta ventas con búsqueda, comparte por WhatsApp y exporta PDF con vista previa. Incluye cuenta regresiva al sorteo y animación con confetti al revelar el ganador.

Stack y arquitectura

Frontend: Vue 3, Pinia, VueUse, Vite y composables propios; lógica centralizada en talonarioStore; SweetAlert2, jsPDF, canvas-confetti y persistencia en localStorage.
Datos: API Colombia para ubicación; sorteo justo con crypto.getRandomValues.

Pokédex

Pokédex interactiva con álbum, búsqueda de Pokémon y minijuegos para explorar la PokeAPI de forma lúdica.

Vue 3QuasarPokeAPI
Objetivo

Consumir la PokeAPI en una SPA moderna que permita consultar fichas de Pokémon, llevar un álbum de descubiertos y practicar conocimiento con juegos integrados.

Cómo funciona

Explora el listado con búsqueda y perfil detallado de cada Pokémon (tipos, stats, sprites). El álbum registra tu progreso. En el hub de juegos puedes adivinar el Pokémon, entrenar memoria, identificar tipos y repasar estadísticas.

Stack y arquitectura

Frontend: Vue 3 con Composition API, Quasar UI, Vue Router, Axios y Vite; composables reutilizables para la API y el álbum; diseño responsive con navegación inferior.
Datos: consumo de la PokeAPI REST pública.

Software Administrativo — Forza Gym

Sistema de gestión para gimnasio con módulos de clientes, planes, reservas, pagos, inventario y más. Proyecto en desarrollo activo.

Vue 3ExpressMongoDB En desarrollo
Objetivo

Digitalizar la operación de un gimnasio en una plataforma web: centralizar clientes, membresías, reservas, pagos, inventario, ventas y mantenimiento de equipos en un solo sistema administrativo.

Cómo funciona

Arquitectura separada en frontend y backend. El panel permite gestionar sedes, usuarios con roles, clientes (con foto y seguimiento), planes, clases, reservas, pagos, caja, ingresos, inventario, proveedores, ventas y mantenimiento de máquinas. Incluye autenticación JWT, tareas programadas y notificaciones por correo.

Stack y arquitectura

Frontend: Vue 3, Quasar, Pinia, Vue Router, Axios, Vite, ApexCharts y CropperJS.

Estado del proyecto

En desarrollo activo: varios módulos ya funcionan, pero el sistema no está terminado al 100%. Se siguen integrando funcionalidades, puliendo flujos y completando la estabilidad general antes de considerarlo producto final.
